# Mission: Strategic Programming

## Why
AI is reducing the value of purely tactical implementation work. The goal is to become the kind of programmer who can shape ambiguous software work: clarify intent, plan increments, protect codebase health, design interfaces, guide QA, and make technical decisions that compound.

## Success looks like
- Turn vague product or engineering goals into small, reviewable, testable implementation slices.
- Explain tradeoffs in terms of risk, reversibility, delivery flow, and long-term codebase health.
- Design changes that leave future developers with clearer options than before.
- Use AI as a force multiplier for coding while retaining human judgment over direction, constraints, and quality.
- Build a portfolio of lessons, references, and practice artifacts that can be revisited.

## Constraints
- Lessons should be short, practical, and grounded in trusted engineering resources.
- Each lesson should teach one strategic skill and include a tight feedback loop.
- The learning path should assume AI assistance is available for tactical coding.

## Out of scope
- Memorizing language syntax or framework APIs as the primary learning goal.
- Treating "senior" as title-chasing rather than observable engineering judgment.
- Abstract software architecture theory without a path back to day-to-day decisions.
